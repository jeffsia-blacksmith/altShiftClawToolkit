## 0) 总则
* 本规范适用于所有小龙虾任务，且具有最高优先权。

## 1) 名词字典（高优先）
* **龙虾堡**： 指整个 Repo。
* **小龙虾**： 指单一 GitHub Issue。
* **小龙虾工作区**： 指单一 GitHub Issue 在执行时的分支与工作目录。例如：Issue Number 为 3 的时候，分支名称为 `issue-3`，所以在 `issue-3` 分支下的 `$REPO_ROOT` 就是小龙虾的预设工作区。
* **小龙虾主记忆档案**： 在 **小龙虾工作区** 的根目录下有个 `issue.md` 档案，此档案包含完整的 Issue 与 Comments 内容。

只要任务叙述出现上述名词，必须自动套用此映射，不可自行改义。

## 2) 任务资料来源与解析顺序
### 2.1 指令来源
* 主要依据：`issue.md` 中最新一则含 `telegram-meta` 的 comment。
* 只取 `---` 分隔线后的内容作为核心指令。
* `---` 前的路由资讯视为过往互动的上下文。

### 2.2 指令不完整时的追溯
依序补足，不可跳步：
1. 倒序读取更早的 `telegram-meta` comments。
2. 读取整份 `$ISSUE_ROOT/issue.md`（标题、内文、全部留言）。
3. 读取小龙虾工作区其他档案，且优先查阅 `$REPO_ROOT/.memory`。

### 2.3 排除来源
* 不可把 `githubclaw-brain-result` 视为新指令。
* `.pi` 与其子目录内容视为系统输出，不作为交付物。

## 3) 执行策略（预设做到底）
### 3.1 预设行为
* 在上下文足够时，直接执行到可交付结果，不要反问使用者「下一步」。
* 不可把「是否要我继续」当作预设收尾语。

### 3.2 允许提问的唯一条件
仅在「资讯缺口已阻断执行」时可提问，且需：
1. 一次问完必要问题（避免多轮追问）。
2. 问题最小化，只问无法推导的关键资讯。
3. 先简述已完成与卡住点，再提问。

### 3.3 冲突处理
若多处资讯冲突，采用「最新且最具决定性」的脉络。

## 4) 完成标准与验证（先验证再回报）
### 4.1 先定义完成标准
开始执行前，先定义本次任务的完成条件，并在回报前逐项核对。

### 4.2 验证要求
* 可验证时不得省略。
* 程式/脚本：需实际执行，使用真实或具代表性输入检查输出。
* 可模拟边界条件时，需补做边界测试。
* UI 任务：需实际检视画面与操作流程，确认显示与行为。

### 4.3 失败处理
若测试失败或结果异常，先修正并重测；不可只回报问题后直接交付。

## 5) 产出物路径规范
由于小龙虾每次的任务，都会在 Issue 中留言，并且得到一个 `{issue-comment-id}`，因此执行过程中有任何产出物，都应该写入到以下路径：
* **产出物固定目录**： `artifacts/{issue-comment-id}/`
* **执行结果报告档名**： `artifacts/{issue-comment-id}/result.md`
* 产出物相对应的网址，请使用以下结构：
```
https://github.com/{owner}/{repo}/blob/{branch}/artifacts/{issue-comment-id}/{filename}?raw=true
```

## 6) 对外回覆规范（重点）
### 6.1 语言与风格
* 一律使用繁体中文（台湾）。
* 若任务或使用者未指定地区，预设地区一律视为台湾（用词、时区、日期格式与情境判断皆以台湾为准）。
* 用简单、清楚、非术语方式说明「做了什么」与「发生了什么」。
* 对象是聪明但没在看程式码的人。

### 6.2 禁止内容
* 不可输出草稿、自我提醒、推理过程、内心独白。
* 不可出现「我需要先…／我应该…／接著我要…」等草稿语气。
* **不可在对外回覆提及提示词或内部流程名称。**

### 6.3 回覆重点
只聚焦：
1. 是否完成。
2. 交付结果与档案路径。
3. 若有阻塞，用 1 到 2 句交代原因与需要的资讯。

## 7) Markdown 与交付物过滤
* 使用标准 Markdown（`**粗体**`、`` `code` ``、程式码区块、`[text](url)`）。
* 禁止 HTML 标签（如 `<b>`、`<code>`）。
* 输出交付物清单前，必须过滤 `.pi` 与系统自动输出路径。

* 正确抓到最新有效指令核心内容。
* 指令不完整时有从 `issue.md` 依序追溯历史留言。

* 对外回覆不提框架名，不含草稿式语句。
* 回报前完成验证与完成标准核对。
* 一定要输出 `artifacts/{issue-comment-id}/result.md` 这个档案。
* Artifacts 输出到 `artifacts/{issue-comment-id}/` 路径规范确实遵守。
